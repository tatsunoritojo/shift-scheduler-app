## 1. プロダクト概要

Shifree は**小規模事業者向けの無料シフト管理 Web アプリケーション**。
飲食店・小売・小規模チーム (3〜30名) をターゲットに、シフト希望の収集から確定・Google カレンダー反映までを一気通貫で提供する。

### 1.1 解決する課題

| 現状の課題 | Shifree の解決 |
|---|---|
| LINE でシフト希望を個別収集 | ワンタップで全員に通知、カレンダーから提出 |
| Excel で毎月シフト表を手作り | カレンダー UI で視覚的にスケジュール構築 |
| 確定シフトの共有が面倒 | Google カレンダーへ自動反映 |
| 急な欠員対応がバタバタ | 候補者自動検索 + メール通知で補充 |

### 1.2 主要な特長

- **アプリ不要**: ブラウザだけで動作 (PWA 対応)
- **完全無料**: 課金・機能制限なし
- **Google 連携**: OAuth ログイン + Google Calendar 双方向同期
- **マルチアカウント連携**: 業務用とプライベート等の複数 Google アカウントのカレンダーを参照可能
- **マルチテナント**: 1つのシステムで複数店舗を独立管理

## 5. 主要ワークフロー

### 5.1 シフト管理フロー (メインフロー)

```
管理者: 期間作成 (draft)
  │
  ▼
管理者: 期間公開 (open) → 全スタッフへ通知
  │
  ▼
スタッフ: シフト希望提出  ← 自動リマインダー
  │
  ▼
管理者: スケジュール構築 (draft)
  │
  ▼
管理者: 承認依頼 (pending_approval) → 事業主へ通知
  │
  ▼
事業主: 承認 (approved) or 差戻し (rejected)
  │
  ▼
管理者: 確定 (confirmed) → Google Calendar 同期試行
  │
  ▼
[同期結果は ShiftScheduleEntry 単位で追跡]
  ├─ 成功: calendar_event_id 記録
  ├─ 認証切れ: sync_error = CREDENTIALS_EXPIRED → スタッフが手動再同期
  └─ API失敗: sync_error 記録 → リトライ可能
```

### 5.2 欠員補充フロー

```
管理者: 欠員リクエスト作成
  │
  ▼
システム: 候補者自動検索 (週間勤務時間の少ない順)
  │
  ▼
管理者: 候補者へ通知メール送信 (承諾/辞退リンク付き)
  │
  ▼
候補者: 承諾 or 辞退 (先着1名、レースコンディション防止)
  │
  ▼
シフト自動入替 + カレンダー再同期 + 変更履歴記録
```

### 5.3 招待フロー

```
管理者: 招待作成 (メール指定 or 招待コード/QRコード)
  │
  ▼
新メンバー: 招待リンクアクセス → Google ログイン
  │
  ▼
自動: 組織参加 + ロール割当
```

## 6. データ規模の想定

| 項目 | 想定値 |
|---|---|
| 1組織あたりのスタッフ数 | 3〜30名 |
| 1期間あたりの日数 | 28〜31日 |
| 月間シフト提出数 | スタッフ数 × 1 |
| 月間スケジュールエントリ | スタッフ数 × 20〜25日 |
| 非同期タスク/日 | 〜50件 (メール + カレンダー同期) |

## 7. 制約事項

| 制約 | 内容 |
|---|---|
| Vercel Hobby プラン | Cron は1日1回のみ (9:00 UTC) |
| サーバーレス | 長時間実行不可 → DB キューで非同期化 |
| ファイルシステム | 読み取り専用 (SQLite は開発のみ) |
| Google API クォータ | Calendar API: 1,000,000 requests/day |
| セッション有効期限 | 24時間 |

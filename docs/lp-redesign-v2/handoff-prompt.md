## プロンプト本文（ここから下をコピペ）

あなたには **シフリー（Shifree）** という無料シフト管理ツールの新しいランディングページ（LP v2）の設計・制作を依頼します。

### このプロジェクトでのあなたの位置づけ

**あなたがデザインの意思決定者です。**

- レイアウト、タイポグラフィ、配色、アニメーション、コピーライティングの最終判断はあなたが行ってください
- 私（発注側）と PM 補佐の Claude は、方向性の合意・技術制約の確認・アウトプットのレビューを担当します
- 本ブリーフの内容に **制約変更の提案があれば歓迎**します（理由付きで）。特に「ここを固定すると craft が出にくい」という指摘があれば、構成レベルでの再提案も OK です
- 迷ったら「craft signal の最大化」「既存 LP との差別化」を優先基準にしてください

### 本件の核心的な要求

LP は **プロダクトのプレビュー** として機能します。つまり：

> **「このくらいの LP を用意できる人が作るプロダクトなら、信頼できそう」**

と、アクセス後 **1 秒** で思わせることが最大の勝負。この印象を作れない LP は失敗扱いです。

### 最初にお願いしたいこと（Phase 0）

以下 3 点を **提案書（A4 数ページ相当 or 同等の Markdown）** としてまとめてください：

1. **ムードボード** — 参考サイト・ビジュアルリファレンス 5〜10 点、テキストで方向性を言語化
2. **Hero コンセプトスケッチ** — 組成アニメの動きをテキスト or 簡易モックで説明。どの craft anchor を軸にするか
3. **接続部（ズームアウト型）の設計アイデア** — マクロ（1 ヶ月体験）から ミクロ（機能深掘り）への視覚的接続をどう実装するか

この段階で **色（§8 の A/B/C のどれ、または独自案）** と **主要タイポグラフィ** も決めておいてください。確定版ではなく「これで Phase 1 に進みたい」という提案レベルで OK です。

Phase 0 の提案を私と PM 補佐でレビューし、合意が取れたら Phase 1（Hero + 接続部の実装）に進みます。

### 大事なこと

- **AI が作ったとバレる LP は失敗**です。`brief.md §5` の禁止リストを必ず読み込んで、避けてください
- 既存 LP は典型的な AI 生成 LP です。「Tailwind blue + lucide アイコン + Pain→Solution + 3 カラム Features + ストック感のあるモック」を全部踏んでいます。これらを 1 つでも踏襲したら失敗扱いです
- 実運用データ（ユーザー数・導入実績）はまだありません。数字で殴れないので、**作り込みそのもので信頼を作る**必要があります
- 開発者個人の露出は不可です。顔・名前・経歴は出さない方針
- 実画面スクショには個人情報が含まれていますが、公開前に一括差し替えする前提です。作業中はそのまま使って OK

### 質問・提案の受け付け方

- ブリーフの前提に納得できない点 → 理由を添えて代案を提案してください
- 制約の中で成立しづらい場合 → 「この禁止を外せばこう作れる」等の交渉を歓迎します
- 素材が足りないと感じた場合 → 具体的にリクエストをください（Admin/Owner 画面の追加撮影など、対応可能）

それでは、Phase 0 の提案をお願いします。

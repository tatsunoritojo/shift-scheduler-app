既存 LP は、テンプレートの語彙で出来ている。Tailwind blue、3 カラム、挑発ヘッドライン。これらを 1 秒で読み取られた瞬間、プロダクトの信頼は目減りする。

本提案では 紙面的な静けさ と 動きの精度 を交換レートにする。明朝の骨格、朱の一点、スクロール連動の組成アニメ。色ではなく、手数で信頼を立ち上げる。

01Position

立ち位置の宣言— Why this direction.

既存 LP のアンチパターンを避けるだけでは不十分。「明朝 × 紙 × 朱 × 精密な動き」という、SaaS 界隈で誰もやっていない組み合わせに賭ける。craft signal は「珍しさ」ではなく「一貫性」から生まれる。このセクションはその賭けの地図。

シフリーのコア差別化は「データが Google に残る」。これは静かで、主張の弱い事実だが、誠実な態度の象徴でもある。派手な SaaS 語彙（インディゴ、グラデ、3D イラスト、絵文字）はこの態度と相性が悪い。

代替として選んだのが 編集的 (editorial) な語彙。雑誌の扉、設計図面の余白、活版の朱の入り方。ターゲットである個人経営事業者・塾・美容室オーナーは、派手なテック文脈よりも、落ち着いた誌面の品に反応する層が一定数いる。

そして編集的な紙面は、スクロール連動の動きと結合したとき、急に現代化する。止まっていれば古いが、精密に動けば「作り込んだ」になる。これが本提案の骨子。

AVOID — 既存 LP のアンチパターン

  • #3B82F6 Tailwind blue プライマリ
  • 青 → 紫のグラデーション背景
  • lucide 線画アイコン 3 カラム並び
  • 「まだ手作業ですか？」型の挑発ヘッドライン
  • 中央寄せ Hero（見出し＋サブ＋CTA の定型）
  • Pain → Solution の二項対立
  • ストック系スマホ mockup
  • 星 5 つの偽レビュー

CRAFT ANCHOR — 選択理由

3 つの candidate のうち採用したのは "Field"（7×n グリッドそのものを Hero の主役にする）。理由は 3 つ。

① 再利用できる：同じグリッドが接続部・1ヶ月体験・機能深掘りに持ち越せるため、LP 全体を 1 つの構造物として閉じられる。

② 「プロダクト = シフト表」を 1 秒で伝える：言語に頼らず、形状で製品カテゴリを示せる。SEO 流入（「シフト管理 無料」）でも誤解が起きない。

③ 色を抑えても動きで解ける：朱と黒だけで意味を分けられ、アンチパターン色（Tailwind blue）を使わずに済む。

REJECTED — 検討したが採らなかった候補

① 文字組成型：「月末が少しだけ早く終わる」の文字がタイプライター風に立ち上がる。craft は出るが、シフト管理と関係がない文字遊びになる。却下。

② 実画面のマウスカーソル誘導型：既存 LP も似た方向。差別化が弱い。却下。

③ 動画（MP4）Hero：重量増・サーバ依存・CLS リスク。LCP 2.5s を守れない。却下。採用する Field は全体 ~38KB、CSS アニメのみで完結。

04Connector

MECHANIC — どう作るか

実装は scroll-timeline（CSS）+ position: sticky が本命。5 フレームを 1 つの sticky container に重ね、scroll の進捗 t ∈ [0,1] を CSS 変数に流し込み、transform: scale(var(--s)) translate(var(--x),var(--y)) を各フレームに適用する。

フォールバック：scroll-timeline 非対応ブラウザでは IntersectionObserver で段階切り替えに退行。「動きが無くても読める」を基準に組む。

接続 B（Section1 → Section2）はこの逆再生。最終フレームの「1 セル」から再度 1× に引いた瞬間、視点が機能深掘りの UI の上で止まる。

SIGNATURE — 共通する hinge セル

全フレームで同じ 「05/14 火曜」のセルを hinge にする。Hero で初登場し、接続部 A の着地点になり、1ヶ月体験 6 場面の中で何度か言及され、接続部 B で同じセルから機能画面へ抜ける。

繰り返しの小ネタではなく、「LP 全体が 05/14 火曜を扱った 1 本の物語」として設計する。ユーザが明示的に気付く必要はない。印象の一貫性だけが残ればよい。

06Open

Phase 1 前に決めたい 6 点— Your calls.

下記は ClaudeDesign 判断で決めず、発注側の合意が欲しい点。それぞれ「放置時の私の既定値」も添える。合意が取れなければ既定値で Phase 1 に着手する。
1 — 既存 UI の濃紺ヘッダー／青アクセントを LP 内では捨てて良いか？
提案では LP 上のスクショも朱系に再塗装する前提。プロダクト実機と LP のブランド色がズレるが、craft signal を優先した判断。既定値: 捨てる（再塗装）。
2 — ロゴの「シ」モチーフを LP 上でどこまで出すか？
masthead 左にだけ小さく置き、Hero には出さない方向。現行ロゴは青（#1266D3）だが LP 上では黒 or 朱のバージョンに置き換えたい。既定値: 黒に置き換え、alt ロゴを 1 本提案。
3 — Hero コピー「月末が、少しだけ早く終わる。」で進めてよいか？
観察型短文。ネガを言わず、具体の量感を出す。代替候補: 「希望を置く。配置が戻る。」／「Shift, free.（主コピーを英語に）」。既定値: 「月末が、少しだけ早く終わる。」。
4 — 使い方動画（MP4）を LP に埋める？
Hero は CSS のみで組むため、動画は「機能深掘り」末尾で任意再生に置く案。重量／CLS リスクあり、遅延ロード前提。既定値: Section 2 内に lazy 埋め込み、Hero には使わない。
5 — Admin / Owner 画面の追加スクショ、撮ってもらえるか？
機能深掘り 8 項目中 3–4 項目（承認プロセス、欠員募集、確定→同期の結果画面）は Admin 側。Worker だけだと主役が偏る。既定値: Phase 2 着手前までに 3 枚依頼。
6 — 公開前の架空データ差し替え、誰がやるか？
「Onedrop」「久保田彩未」を「晴ヶ丘塾」「佐藤陽菜」等に差し替える作業。LP 側で配置して、原稿は発注側で固定、の分担が速い。既定値: ClaudeDesign が叩き台配置、発注側で最終確定。

## プロンプト本文（ここから下をコピペ）

Phase 1 成果物、受領しました。**構造 craft は高く評価している** ため、まず先にこれを伝えておきます：

- hinge セル「05/14 火」を LP 全体の motif とした設計
- 7×5 グリッドの Hero / 接続部再利用
- 9 秒ループの組成アニメ timing（wish hatch → chip → hinge zoom）
- scroll-timeline + IntersectionObserver フォールバック
- 明朝 × mono × display の 4 本柱タイポグラフィ

これらは **保存してほしい資産**です。

### 依頼の主旨 — 色パレットのみ再提案

LP 外のブランド意思決定が入りました。**LP 単独の craft 最適化ではなく、App UI も含めた統一ブランド刷新** という上位レイヤの制約が追加されます。

#### 変更の動機（あなたの責任ではない）

- 既存 App UI は濃紺 + 青アクセントで組まれている
- Phase 0 で「LP 側で朱に振る」を承認したが、**LP → App 遷移時のブランド破綻**が実運用上の問題として顕在化
- Onedrop パイロット（〜2026-05-03）中でユーザー数 12 名。ブランド移行コストが歴史的最小の今、LP と App を同じパレットに統一する判断

したがって Phase 1 の朱方向は、**craft として正しかったが、ブランド制約で却下** となる。これは前回判断の失敗ではなく、外部条件の変化です。

#### 新パレット（確定）

| 役割 | 名称 | Hex | CMYK |
|---|---|---|---|
| メイン | BRIGHTBLUE | `#00CCFF` | 100.20.0.0 |
| アクセント | YELLOW | `#FFE626` | 0.10.85.0 |

位置づけ: 既存 LP / App の **「青というブランドシグナル」を継承しつつ、執行を craft 化** する方向。既存の `#3B82F6`（Tailwind blue）や `#1266D3`（App 濃紺）は AI テンプレ / デフォルト色なので捨てる。新しい `#00CCFF` は同じ「青系」だが全く別物の craft signal を発する色。

### Phase 0 禁止リストの更新

以下の 3 項目は **削除** します（色による過剰 NG）：

- ~~Tailwind blue `#3B82F6` 禁止~~
- ~~青 → 紫、青 → シアングラデーション禁止~~
- ~~`#6366f1`, `#8b5cf6` 等のインディゴ/バイオレット禁止~~

代わりに **新規追加** する NG：

- BRIGHTBLUE `#00CCFF` を Tailwind blue `#3B82F6` と近接使用する配色
- YELLOW `#FFE626` を警告色として使う（アクセント専用、注意喚起は別色で）
- BRIGHTBLUE の多階調グラデーション（単色規律を保つ）

**色以外の禁止リストは brief.md §5 の通り継続**。挑発ヘッドライン、絵文字、3 カラム Features 等、全て生きています。

5. **Open Questions への既定値** — 下記 5 問

### 新 Open Questions

以下 5 問について **ClaudeDesign の既定値を提示** してください。発注側が合意できなければ調整する流れです：

1. **配色方針** — 以下のどれを軸にする？
   - ① 地=paper（温白）+ BRIGHTBLUE メイン + YELLOW アクセント（編集誌面系）
   - ② 地=BRIGHTBLUE + 白文字 + YELLOW スパーク（Blue Note / RISO 系）
   - ③ 地=ink（墨）+ BRIGHTBLUE 主 + YELLOW サブ（モード系）
   - または独自案

2. **YELLOW の役割定義** — 純粋アクセント / データ可視化 / リンク hover / etc のどれ（または併用）

3. **hinge セル「05/14 火」のハイライト色** — 旧 = 朱。新 = BRIGHTBLUE / YELLOW どちら（または併用）

4. **Hero 組成アニメの wish hatch 色** — 旧 = 朱のハッチ。新 = どの色・どのパターン

5. **明朝タイポは継続で良いか** — 継続を推奨するが、BRIGHTBLUE 基調だと sans-serif へのシフトも選択肢になり得る

### Phase 1 成果物の継承ポリシー

- **保存して色替えで済ませるもの**: 構造 craft 全般（上記の冒頭リスト）
- **差し替えるもの**: `:root` の色変数 9 本 / 透過 rgba ハードコード / スクショ色調
- **あなたの判断で「色と一緒に変えたほうが良い」構造要素があれば**、根拠付きで提案してください。例: 「BRIGHTBLUE だと wish hatch の斜線角度は 45° より 30° のほうが craft signal が立つ」等

### 大前提（繰り返し）

- LP を 1 秒で「他と違う」「信頼できる」と感じさせること
- 既存 LP / App の default 色っぽさに戻らないこと（BRIGHTBLUE は「default 青」ではなく「craft 青」として立てる）
- AI 臭いパターン（絵文字、3 カラム、挑発コピー等）は引き続き全面禁止

特に急ぎではありません。品質優先。完成したら HTML 一式で送ってください。

質問・提案があれば作業前に投げてください。

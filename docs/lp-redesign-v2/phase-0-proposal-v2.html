01Stance

青に戻る、ただし既定の青には戻らない— Why #00CCFF, not #3B82F6.

#00CCFF は、SaaS 界隈で「業務ツールの青」として流通していない明度・彩度。シアン寄りの鮮烈さは印刷（RISO）や Blue Note のレコード盤に近く、テンプレ青から語彙的に離れている。BRIGHTBLUE を 地の色として大面積で使う こと自体が、テンプレの「白地 + 青アクセント」から離れる craft signal になる。

Phase 1 の朱方向は craft として正しかったが、LP → App 遷移でブランドが破綻するという外部制約で却下された。新方針は「青というブランドシグナルを継承しつつ、執行を反転させる」。反転の仕方は、量と比率で決める。

テンプレ青の執行は「白地 70%、青アクセント 10%、濃紺ヘッダー 5%」。BRIGHTBLUE の執行は逆に、青地 50%、白紙 35%、YELLOW 3%、墨 12%。青を少量のアクセントではなく、紙そのものとして扱う。

明朝タイポの継続はこの量比で効く。ゴシックの青地は IT ポスターに収束するが、明朝の青地は Blue Note のアルバムジャケットに寄る。この寄り先が、競合 SaaS との距離になる。

COLOR CHOICES — 各パートに付いた色の理由

地 = BRIGHTBLUE：SaaS の定型は「白地 + 青アクセント」。これを反転させる。青を地にする瞬間、LP は広告ポスターとレコードジャケットの語彙に接続する。

wish ハッチ = 白 · 30°：朱の 45° は紙に沈むが、青地の白 30° は透過する RISO 版の重なりを暗示する。角度は 30° を採用（45° だとフラッグ感が出る）。

assigned チップ = YELLOW：3% 未満の面積で視線軸を引く。Tailwind の青 + 緑チェックでは出せない「読む紙面」の強度がここで立つ。

hinge = YELLOW 抜き：朱版のリングは「囲み」、YELLOW 版は「抜き」。セルそのものを YELLOW で塗り、04/14 火だけが発光する。

HERITAGE — Phase 1 から持ち込むもの

9 秒ループの timing、4 段の axis（FIELD → WISH → ASSIGN → ZOOM）、hinge セル (r2, c1) の位置、SKIP ボタン 3 段（再生 / 一時停止 / 静止）、IntersectionObserver による offscreen pause。全て変更なし。

変わるのは CSS の :root 変数 9 本と、透過 rgba の色相値のみ。実装としては 色パッチ適用の規模で完了する。

07Open · Defaults

新 Open Questions への既定値— Pending your call.

addendum §6 の 5 問に対する既定値。合意が取れなければここで調整。取れれば Phase 1.5（色パッチ適用）に即着手する。
Q1 — 配色方針
既定値: ② 地=BRIGHTBLUE + 白紙面ゾーン + YELLOW スパーク。①（白地主）は craft signal が弱い（既定の SaaS 執行との距離が 2 割程度）。③（墨地）は夜間誌面として強いが、LP の多くの時間を占める Scene 1〜6 のスクショが暗所で沈む。②は Hero と接続部で青を大面積に使い、本文ゾーン（Scene 1〜6）は paper に降ろすことで、スクショ読みやすさとブランド主張を両立する。
Q2 — YELLOW の役割
既定値: 純粋アクセント + hinge motif の 2 役のみ。データ可視化には投入しない（ブランド色をデータに使うと記号疲弊が起きる）。リンク hover には使わない（brand-deep の下線で処理）。警告色にも使わない（addendum §3 新 NG）。総面積は画面あたり 3% 以下を規律として持つ。
Q3 — hinge セル「04/14 火」のハイライト色
既定値: YELLOW。BRIGHTBLUE をハイライトに使うと、青地との対比が立たない（同系色）。hinge は「青の海の中で唯一 YELLOW」という図式で、1 秒で視線軸を作る役を担う。リングではなく 抜き（セルそのものを YELLOW で塗る）を採用し、接続部のズームイン時は黄地の中に UI が立ち上がる構成に。
Q4 — Hero 組成アニメの wish hatch 色
既定値: 白（paper-on-blue #F5F3EC）· 30° · 透過 0.85。45° は旗・帯のシンボルに寄るため避け、30° で版ズレの粒子感を出す。YELLOW ハッチは assigned チップと色が被るため却下。青の濃淡ハッチは craft signal が弱く、addendum §3 の「単色規律」にも抵触するため却下。
Q5 — 明朝タイポ継続可否
既定値: 継続（Shippori Mincho）。BRIGHTBLUE + 明朝は Blue Note レコード盤の語彙を直接継承でき、本件が目指す「1 秒で他と違う」に最短で到達する。Sans-serif 化すると青地 + Helvetica 的な一般 IT ポスターに収束し、差別化が消える。ただし 青地上の weight だけ 600→700 に上げる（光量補正、CSS 側で自動切り替え）。

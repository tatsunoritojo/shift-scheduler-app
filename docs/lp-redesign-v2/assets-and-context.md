## 3. 競合と差別化軸

### コア差別化ポイント: **「データが Google に残る」セキュリティモデル**

シフリーは、シフトデータの実体を Google カレンダーに書き込む設計。自社 DB には予定の「生データ」ではなく、参照情報とメタデータしか持たない（設計思想）。

### これが意味すること（LP で訴求可能な文脈）

1. **事業者がシフリーの利用をやめても、データは Google に残る** — ベンダーロックインがない
2. **セキュリティ責任の大半を Google 側に移譲できる** — 個人事業主が自力でセキュリティを気にしなくてよい
3. **漏洩リスクの極小化** — 仮にシフリー側 DB が漏洩しても、業務データの核心は Google 側にある
4. **BCP 観点** — シフリーがダウンしても、確定シフトは各 Worker の Google カレンダーで閲覧できる

### 主要競合と比較して強いポイント

| 競合 | 一般的な特徴 | シフリーの優位性 |
|---|---|---|
| Airシフト、FREENANCE | 独自 UI + 独自 DB。データはベンダーに蓄積 | データ主権がユーザー側 |
| HRBrain シフト管理、シフオプ | 多機能・重量級・有料 | 無料 + 必要最小限の機能で学習コスト低 |
| LINE グループ + Excel | 無料だがスケールしない | 自動化で運用コスト削減 |
| Google スプレッドシート手作り | 柔軟だが手間 | スプレッドシート + Google カレンダーの利点を自動化で接続 |

### LP コピーへの応用

「データを Google に置く」という事実は、次のような切り口で craft に昇華できる：

- **セキュリティ文脈**: 「シフトデータは Google にあります。あなたのアカウントに。」
- **自由度文脈**: 「いつでも抜けられます。データはあなたのものです。」
- **信頼文脈**: 「Google が持つセキュリティを、そのまま使います。」

※具体的なコピーは ClaudeDesign 判断だが、この **"ベンダーロックしない" スタンス** は Hero で訴求する価値あり。

## 4. 運用コンテキスト（LP 公開後の想定）

### 流入経路（推定）

| 経路 | CTA の作り方 |
|---|---|
| 口コミ（事業者同士） | 登録よりも「まず見せる」設計、1ヶ月体験セクションが主役 |
| Google 検索（「シフト管理 無料」等） | 既存 LP の SEO 資産（JSON-LD、meta）を継承 |
| SNS・note 等の記事経由 | OG 画像と Hero の 1 秒印象が重要 |
| Product Hunt / Zenn / Qiita | 開発者向けにも刺さる craft が効く |

### 想定シーン像

- **飲食店店長**（30-40 代男女）— 10-15 名のアルバイト、毎月シフト作成に半日以上かけている
- **美容室オーナー**（30 代前後）— 5-8 名、LINE で希望集めて Excel で組み立て
- **塾・教室経営者**（30-50 代）— 10-20 名、学生バイトが多く入れ替わり激しい ← **Onedrop 塾が典型**
- **小売店マネージャー**（店長職）— 本部の承認が必要、権限分離に価値を感じる

### Do / Don't（LP コピー方針）

**Do**
- 「Google に残す」という設計思想の静かな提示
- 「無料」の明言（条件付きでないこと）
- 自動化のビフォーアフター（LINE/Excel → シフリー の対比は OK、ただし悪く描きすぎない）

**Don't**
- 競合製品の名指し dis（Airシフト、HRBrain 等を直接批判しない）
- 「AI」「自動化」という言葉の多用（安っぽくなる）
- 「累計○人が利用」等、実績未達の数字（育っていない）
- 「導入事例」セクション（顧客公開承諾を取っていないため）
- 開発者個人（東城さん）の顔・名前出し（本人意向により NG）

## 5. ブランド語源 / ドメイン

### 名前の由来
**シフリー = Shift + free**。ストレートに「シフトからの自由」を意味する。

LP コピー活用例（参考、ClaudeDesign 判断で採否）：
- 「シフトを、少し自由に」
- 「Shift, free.」
- サブブランドや section タイトルで Free の二重意味（自由 × 無料）を織り込む

### ドメイン
- 現状: `shifree.vercel.app`
- 今後: 独自ドメイン取得予定あり（時期未定）
